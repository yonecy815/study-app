# 先生システムの再設計

## 🎯 変更の目的

現在の設計では問題が生徒に直接紐づいているため、先生が問題を一元管理できません。
新しい設計では、先生が問題を管理し、生徒ごとに問題を割り当てる方式に変更します。

---

## 📊 新しいデータフロー

### 従来の設計（問題あり）
```
先生 → 生徒 → 問題（生徒に直接紐づく）
```
- 先生は生徒ごとに問題を作成する必要がある
- 同じ問題を複数の生徒に使い回せない
- 問題の一元管理ができない

### 新しい設計（改善）
```
先生 → 問題（先生に紐づく）
    ↓
    割り当て（student_problems）
    ↓
    生徒のフォルダに表示
```
- 先生が問題を一度作成すれば、複数の生徒に割り当て可能
- 問題の一元管理が可能
- 生徒ごとにカスタマイズした学習内容を提供

---

## 🗄️ データベース構造

### 新しいテーブル: `student_problems`

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGSERIAL | 主キー |
| student_id | TEXT | 生徒のID |
| problem_id | BIGINT | 問題のID（problemsテーブルを参照） |
| assigned_at | TIMESTAMP | 割り当て日時 |

**ユニーク制約**: `(student_id, problem_id)` - 同じ問題を同じ生徒に重複して割り当てない

---

## 🔄 データの流れ

### 1. 先生が問題を作成
```sql
INSERT INTO problems (user_id, question_text, answer_text, subject, ...)
VALUES ('先生のID', '問題文', '答え', '科目', ...);
```

### 2. 先生が生徒に問題を割り当て
```sql
INSERT INTO student_problems (student_id, problem_id)
VALUES ('生徒のID', '問題のID');
```

### 3. 生徒が自分の問題を取得
```sql
SELECT p.*
FROM problems p
INNER JOIN student_problems sp ON p.id = sp.problem_id
WHERE sp.student_id = '生徒のID';
```

## 🖥️ UIの変更

### 先生モードのメニュー画面

**追加するボタン:**
- 📥 **問題を追加** - 先生自身の問題を作成・管理

**既存の機能:**
- 👥 生徒を選択 - 生徒ごとの編集画面へ

### 先生の問題管理画面（新規）

- 問題の一覧表示
- 問題の追加（CSV / 手動入力）
- 問題の編集・削除
- フォルダ別フィルタ
- 検索機能

### 生徒の編集画面（変更）

**従来:**
- 問題を直接アップロード

**新しい:**
1. **先生の問題一覧を表示**（チェックボックス付き）
2. **割り当てたい問題を選択**
3. **「反映」ボタンで生徒に割り当て**
4. **割り当て済みの問題を表示**（解除も可能）

### 生徒の自主学習画面（動作の変更）

**従来:**
- `problems` テーブルから直接取得

**新しい:**
- `student_problems` テーブル経由で取得
- 先生が割り当てた問題のみ表示

### ステップ2: 既存データの移行（オプション）

既存の生徒の問題を新しいシステムに移行する場合：

```sql
-- 既存の生徒の問題を student_problems に移行
-- 注意: user_id が生徒IDの問題のみ
INSERT INTO student_problems (student_id, problem_id)
SELECT user_id::text, id
FROM problems
WHERE user_id IN (SELECT id FROM profiles WHERE teacher_id IS NOT NULL);
```

## ⚠️ 注意事項

### データの整合性
- 先生が問題を削除すると、生徒への割り当ても削除される（CASCADE）
- 生徒が削除されても、問題は残る

### パフォーマンス
- `student_problems` にインデックスを作成済み
- 大量の問題でも高速に動作

### 既存機能との互換性
- 既存の学習履歴（history）は維持
- フォルダ機能は引き続き使用可能
